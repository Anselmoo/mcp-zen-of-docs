# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for security problems.

Instead:

1. Use GitHub Security Advisories if available for the repository.
2. If that is not available, contact the maintainers privately with:
   - a clear description of the issue
   - impact and affected versions
   - reproduction steps or a proof of concept
   - any suggested mitigation

We will acknowledge the report, investigate it, and coordinate disclosure once a fix is ready.

## Scope

Please report vulnerabilities related to:

- the MCP server runtime
- generated documentation artifacts that create security risk
- workflow or packaging issues that could affect users

## Supported versions

Security fixes are generally applied to the latest supported release on the default branch.
